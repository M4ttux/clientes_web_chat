# Supabase
https://supabase.com/

Supabase es un backend as a service (BaaS), que se presenta a si mismo como una alternativa a Firebase (otro BaaS, pero de Google).
Como BaaS, Supabase incluye las siguientes funcionalidades:
 - Incluye una base de datos PostgreSQL.
 - Incluye funcionalidades y sincronizaciones en tiempo real.
 - Incluye autenticación y autorización.
 - Incluye la capacidad de almacenar archivos.

La idea de un BaaS es que nosotros no tengamos que programar nuestro backend entero, sino que trabajemos mayormente con un backend pre programado que podemos configurar, y con el que podemos interactuar mayormente desde nuestro frontend.

El gran beneficio que tiene este tipo de practica es que permite iterar sobre nuestro proyecto muy rápido.
La contra es que tenemos menos control del backend. Si bien podemos crear funciones en Supabase para personalizar el comportamiento de algunas cosas, es limitado con respecto a lo que podemos hacer en un backend propio.

## Supabase vs Firebase

Firebase es tal vez el BaaS mas conocido, ya que es propiedad de Google, y ofrece una amplia gama de funcionalidades.
Supabase, como dijimos, se presenta como una alternativa "open-source" de Firebase.
Hay algunas diferencias entre los dos:
 - Firebase usa una base de datos NoSQL propia. Supabase utiliza PostgreSQL.
 - Firebase es 100% propietaria, y solo se puede usar a traves de la nube de Google. Supabase es "open-source", y si bien tiene un hosting en la nube propio (que es el que vamos a usar), permite que ustedes puedan auto-hostear el BaaS en su propio servidor.
 - Usar Firebase implica un "vendor lock-in", mientras con Supabase no.

## Base de datos

Firebase usa una base de datos NoSQL propietaria, que tiene algunas similitudes con MongoDB (es orientada a documentos), pero bastante mas limitada.
Supabase usa PostgreSQL, que es una base de datos relacional.
Esto implica algunas cosas:
 - Tenemos mucha mayor libertad en Supabase de que hacer con nuestro datos. como es una base SQL, nosotros podemos fácilmente exportar (hacer un dump) nuestra base de datos, e incluso llevar a un backend diferente. En Firebase, como es propietaria, no podemos simplemente migrar la base de datos "tal cual" a otro servidor. y peor aun, Firebase *no tiene un mecanismo de exportación de su base de datos*.
 - Las bases NoSQL son mas famosas por su facilidad de uso, pero suelen ser menos robustas que las SQL, y muchas mas limitadas a la hora de relacionar información. Como la de Firebase, ademas, es propietaria, uno tiene que aprender a usarla específicamente, y los conocimientos adquiridos de hacer esto no necesariamente se migran bien a otro sistema. Supabase, al usar una base SQL, permite aprovechar todos los beneficios de SQL (robustez, integridad referencial, seguridad, etc), nuestro conocimientos previos de SQL nos ayudan para manejar la base de datos, y cualquier cosa que aprendemos de aca nos va a servir para usar PostgreSQL en cualquier otro entorno.

 ### Opciones de hosting
 Los BaaS nos permiten iterar y desarrollar prototipos o proyectos muy, y suelen incluir tiers gratuitos muy generosos. pero a la hora de tener que subir a producción nuestro proyecto, si el mismo tiene éxito, vamos a necesitar empezar a pagar. y los costos de operación de un baas pueden rápidamente irse de las manos.
 Con Firebase, no tenemos mas alternativa que pagar l oque google imponga, o tener que migar todo nuestro backend, lo que implica tener que hacerlo de nuevo.
 Con Supabase siempre vamos a tener la opción de poder auto-hostear el backend. de esta forma, evitamos cualquier pago adicional por su uso y solo pagaríamos nuestro propio servidor. e incluso si quisiéramos migrar fuera de Supabase, como la base es PostgreSQL, mucho de lo que ya tenemos hecho nos puede seguir sirviendo.

 ### "Vendor Lock-In"

 LLamamos asi a la situación en donde quedamos atados a un determinado proveedor "vendor" para nuestro proyecto, ya que cambiarlo implica tener que hacer todo de nuevo.
 Usar Firebase implica un lock in a esa plataforma, ya que todo es propietario.
 Supabase, al ser open-source, no tine es te problema.
